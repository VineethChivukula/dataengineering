# Data Architecture Overview

I have already covered the definition of Data Architecture in the [Data Engineering Overview](/2.%20Intro%20to%20Data%20Engineering/01_data_engineering_overview.md) section. I want to discuss it in a little more detail.

Take any project, and you will realize that the sole purpose of it is to create value for the end-users. The end-users are called business users in the data world. The project solves business problems that the business users have and we data engineers will take the responsibility of building the data infrastructure to solve those problems.

Okay, that's great, but we do not code or build something right away. I mean, will you start building a house without a blueprint? No, you will not. So, we need some kind of blueprint to proceed with the project. This blueprint is called Data Architecture.

Data Architecture involves more than just designing. It encompasses the entire process of creating and managing the data infrastructure, including data modeling, data storage, data processing, and data governance. It ensures that the data is organized, accessible, and secure, while also aligning with the business goals and requirements.
