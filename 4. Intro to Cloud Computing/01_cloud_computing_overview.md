# Cloud Computing Overview

A few years ago, companies had to invest heavily in physical hardware and infrastructure to run their applications and services. Apart from the initial capital expenditure (Money required to invest for hardware and infrastructure), they also had to deal with ongoing maintenance, upgrades, and scaling challenges.

Some intelligent people proposed that instead of owning and maintaining physical servers which is a pain in the aah, there should be a way to access computing resources over the internet. This idea led to the birth of cloud computing.

## Cloud Computing Definition

Cloud computing is the delivery of computing services which includes servers, storage, databases, networking, software, analytics, and intelligence over the Internet via a pay-as-you-go pricing model. It allows businesses to access and use computing resources on-demand without the need for upfront investments in hardware or infrastructure.

## On-Premises vs Cloud Computing

The following table compares the key differences between on-premises infrastructure and cloud computing:

| On-Premises                                                                                          | Cloud Computing                                                                                                                            |
| ---------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| Requires significant upfront capital expenditure for hardware and infrastructure.                    | Pay-as-you-go pricing model, allowing businesses to pay only for the resources they use.                                                   |
| Maintenance, upgrades, and scaling are the responsibility of the business.                           | Cloud service providers handle maintenance, upgrades, and scaling, allowing businesses to focus on their core activities.                  |
| Limited scalability, as businesses may need to invest in additional hardware to accommodate growth.  | Highly scalable, allowing businesses to easily scale up or down based on demand without the need for additional hardware investments.      |
| Limited accessibility, as resources are typically only accessible within the organization's network. | Accessible from anywhere with an internet connection, enabling remote work and global collaboration.                                       |
| Higher risk of downtime and data loss due to hardware failures or disasters.                         | Cloud providers typically have robust infrastructure and disaster recovery measures in place, reducing the risk of downtime and data loss. |

## The Shared Responsibility Model

In cloud computing, the shared responsibility model defines the division of responsibilities between the cloud service provider and the customer. To be simply put, the cloud provider is responsible for the security of the cloud, while the customer is responsible for the security in the cloud. This means that the cloud provider is responsible for securing the underlying infrastructure, while the customer is responsible for securing their applications, data, and access controls within the cloud environment.
The amount of responsibility that falls on the customer can vary depending on the type of cloud service model. We will discuss these in the upcoming sections.
